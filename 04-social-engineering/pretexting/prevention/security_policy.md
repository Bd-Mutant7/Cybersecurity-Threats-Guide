## 🎯 Purpose

The purpose of this policy is to establish guidelines and procedures to protect [Company Name] employees, customers, and assets from social engineering attacks. This policy defines the requirements for identifying, preventing, and responding to social engineering attempts.

## 📚 Scope

This policy applies to:
- All employees, contractors, consultants, and temporary workers
- All company facilities, systems, and data
- All communication channels (phone, email, in-person, social media)
- All third-party vendors and partners with access to company resources

## 📖 Definitions

| Term | Definition |
|------|------------|
| **Social Engineering** | The psychological manipulation of people to divulge information or perform actions |
| **Phishing** | Fraudulent emails attempting to obtain sensitive information |
| **Pretexting** | Creating a fabricated scenario to steal information |
| **Vishing** | Voice phishing via phone calls |
| **Smishing** | SMS/text message phishing |
| **Tailgating** | Following authorized personnel into restricted areas |
| **Quid Pro Quo** | Offering a service in exchange for information |
| **Baiting** | Offering something enticing to deploy malware |

## 🛡️ Policy Requirements

### 1. General Security Awareness

#### 1.1 Employee Responsibilities
All employees must:
- Complete annual security awareness training
- Report all suspicious communications immediately
- Never share passwords or credentials
- Verify identity through official channels
- Lock workstations when unattended
- Challenge unfamiliar individuals in restricted areas

#### 1.2 Security Training Requirements
- Initial security training upon hire
- Annual refresher training
- Quarterly phishing simulations
- Monthly security tips and updates
- Specialized training for high-risk roles

### 2. Communication Security

#### 2.1 Email Security
- All emails must be sent via authenticated servers (SPF/DKIM/DMARC)
- External emails must be clearly marked
- Suspicious emails must be reported immediately
- Never click links in unexpected emails
- Never open unsolicited attachments

#### 2.2 Phone Security
- Verify caller identity through official channels
- Never provide sensitive information over the phone
- Document suspicious calls
- Use call-back verification for sensitive requests
- Be wary of caller ID spoofing

#### 2.3 In-Person Security
- Wear ID badges at all times
- Challenge individuals without visible ID
- Escort all visitors
- Report suspicious individuals
- Never prop open secure doors

### 3. Information Classification

#### 3.1 Public Information
Can be freely shared with anyone
- Marketing materials
- Press releases
- Job postings

#### 3.2 Internal Information
Limited to employees
- Internal policies
- Employee directories
- Meeting schedules

#### 3.3 Confidential Information
Limited to authorized personnel
- Financial data
- Customer information
- Strategic plans
- Intellectual property

#### 3.4 Restricted Information
Highly sensitive, limited access
- Passwords/credentials
- Security vulnerabilities
- Legal documents
- Trade secrets

### 4. Verification Protocols

#### 4.1 Identity Verification
Before sharing any sensitive information:
- Verify through at least two independent channels
- Use pre-established verification codes
- Call back using official numbers (not caller-provided)
- Check photo ID for in-person requests
- Verify with supervisor for unusual requests

#### 4.2 Request Verification
For any sensitive request:
- Confirm through official channels
- Get written authorization
- Verify with second person for financial transactions
- Check request against normal patterns
- Escalate unusual requests to management

### 5. Incident Reporting

#### 5.1 Immediate Actions
If you suspect a social engineering attack:
1. **STOP** all communication
2. **DO NOT** provide any information
3. **DOCUMENT** details of the interaction
4. **REPORT** immediately to IT Security
5. **NOTIFY** your supervisor

#### 5.2 Reporting Channels

| Incident Type | Contact Method | Response Time |
|---------------|----------------|---------------|
| Email Phishing | phishing@company.com | 1 hour |
| Phone Vishing | security@company.com | 1 hour |
| In-Person | Call Security: ext. 5555 | Immediate |
| SMS Smishing | security@company.com | 4 hours |
| Social Media | socialmedia@company.com | 4 hours |

#### 5.3 Required Information
When reporting, include:
- Date and time of incident
- Method of contact (phone, email, in-person)
- Caller/sender information
- Description of request
- Any information disclosed (if any)
- Screenshots or recordings (if available)

### 6. Technical Controls

#### 6.1 Email Controls
- SPF/DKIM/DMARC implementation
- Spam and phishing filtering
- Attachment sandboxing
- Link protection
- External email warning banners

#### 6.2 Phone Controls
- Call authentication (STIR/SHAKEN)
- Robocall blocking
- Number blacklisting
- Call logging

#### 6.3 Physical Controls
- Access control systems
- Video surveillance
- Visitor management system
- Intrusion detection

### 7. Social Media Security

#### 7.1 Personal Use
- Do not share work information on personal accounts
- Be cautious of friend requests from unknown individuals
- Report fake accounts impersonating the company
- Never post about security incidents

#### 7.2 Official Company Accounts
- Two-person approval for all posts
- Monitor for impersonation
- Regular password changes
- Secure account management

### 8. Vendor and Third-Party Security

#### 8.1 Vendor Requirements
- Must complete security questionnaire
- Must sign confidentiality agreements
- Must undergo background checks
- Must report security incidents

#### 8.2 Vendor Access
- Limited to necessary systems only
- Time-limited access
- Regular access reviews
- Immediate revocation upon contract end

## 🚨 Incident Response Procedures

### 10.1 Suspected Phishing Email
- 🚫 **DO NOT CLICK** any links  
- 📎 **DO NOT OPEN** attachments  
- 📝 **Report** using the PhishAlarm button (or company equivalent)  
- 📧 **Forward** to `phishing@company.com`  
- 🗑️ **Delete** the email


### 10.2 Suspicious Phone Call
- 🚫 **DO NOT PROVIDE** any personal information  
- 📞 **Ask for a callback number**  
- ☎️ **HANG UP** immediately  
- 🔎 **Call back** using the official company or agency number  
- 📨 **Report** the incident to `security@company.com`


### 10.3 In-Person Suspicious Activity
- 🤝 **BE POLITE but firm**  
- 🚶 **ESCORT** the visitor to reception if necessary  
- ☎️ **CALL security immediately**: ext. 5555  
- 📝 **DOCUMENT** physical description and behavior  
- 📢 **NOTIFY supervisor**  


### 10.4 Data Breach
- 🖥️ **ISOLATE** affected systems  
- 💾 **PRESERVE** evidence  
- 📢 **NOTIFY IT Security immediately**  
- 📝 **DOCUMENT** all actions taken  
- 📋 **FOLLOW** the organization’s incident response plan
